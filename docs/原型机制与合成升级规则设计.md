# 《拼读保卫战》游戏原型机制与合成升级规则设计

本文档基于《牛津自然拼读 (Oxford Phonics World) 1-5级》词汇大纲，针对塔防原型的“去广告机制”以及核心的“拼读合成规则”进行了深度细化。重点解决了不同级别发音规则的交叉重叠问题，确保游戏既具备塔防的策略性，又能精准达到自然拼读的教育目的。

## 一、 原型替代方案：Super Simple Songs 激励机制 (无广告模式)

在原型测试和早期面向儿童的版本中，完全移除商业视频广告，替换为具有教育意义的**“儿歌激励时间 (Song Time Reward)”**。

* **触发条件：** 当玩家遭遇绝境（如基地血量告急）、需要扩充网格（购买铲子/魔法书）、或想要刷新商店道具时触发。
* **表现形式：** 游戏画面暂停，屏幕中央弹出一张精美的 **Super Simple Songs 经典儿歌歌词海报**（例如：*Twinkle Twinkle Little Star, Baby Shark, The Wheels on the Bus*）。
* **倒计时机制：** 海报下方显示 **30秒倒计时沙漏**。在此期间，游戏后台播放对应的儿歌片段。
* **奖励结算：** 30秒倒计时结束后，玩家获得等同于原版看广告的丰厚奖励（如：全屏清怪的橡皮擦战车、解锁2个新网格、获得强力词缀道具）。此设计既起到了延时控制节奏的作用，又让孩子在等待中磨耳朵、认歌词，家长接受度极高。

## 二、 核心塔防合成与升级规则总纲

游戏网格内的单位分为**“基础字母块 (Letter Tiles)”**和**“单词英雄 (Word Heroes)”**。为了降低儿童操作的复杂度并避免自动合成引发的策略崩溃，我们制定以下基础规则：

### 1. 同质升级规则 (基础塔防强化)

**规则：** 两个完全相同且等级相同的“基础字母块”或“字根块”，可以通过拖拽进行合并升级。

* **示例：** 1级 'a' + 1级 'a' = 2级 'a'。
* **效果：** 等级上限为5级。每升一级，该字母块的单体发射速度或基础伤害提升 20%。
* **教育意义：** 让孩子在不断拖拽合并的过程中，反复视觉强化同一个字母或字根。

### 2. 异质拼读合成 (单词英雄召唤)

**规则：** 当网格中存在可以拼成“目标单词”的几个独立的字母块时，将它们按拼读顺序排列或拖拽至一起，即可融合进化为强大的“单词英雄”。

* **合成方式设计 (解决交叉冲突)：** 废弃“全自动合成”（避免玩家原本想凑 cat，结果系统自动把刚刷出来的 c 和 a 变成了别的怪）。采用**“连线/拖拽触发”**：玩家手指从 'c' 划到 'a' 再划到 't'，或者将 'c' 拖入一个特殊的“融合阵”中，确认后才生成【CAT (猫咪刺客)】。

## 三、 各级别 (Level 1-5) 拼读合成规则细化与简化方案

自然拼读规则极其繁杂，直接全部放入召唤池会导致游戏无法进行。我们采用**“字块模块化 (Chunking)”**策略，将复杂的发音规则打包成一个不可分割的整体棋子。

### Level 1: The Alphabet (字母发音阶段)

* **召唤池设定：** 仅掉落26个单字母棋子。
* **合成规则：** 此时不强调单词合成。重点是“同质升级”。
* **特色机制：** A升级到最高级，会自动变成对应的代表单词英雄（如 Apple），B变成 Bear，作为新手期的爽感体验。

### Level 2: Short Vowels (短元音 CVC阶段)

* **痛点：** 掉落 c, a, p, t, b 等单字母，凑单词极其困难。
* **简化规则：Word Family (词族) 整体掉落。**
* **具体操作：** 召唤池不再掉落单个元音，而是直接掉落**“词根块 (Rime)”**，如 **-at, -en, -ig**。
* **合成公式示例：** 玩家只需要召唤出辅音 **'c'** 和词根块 **'-at'**。拖拽 **'c' + '-at' = CAT英雄**。
* **优势：** 极大地降低了凑齐难度，并且符合自然拼读中Onset-Rime（声母-韵母）的教学法。

### Level 3: Long Vowels (长元音与 Magic E 阶段)

* **痛点：** “Magic E” (a\_e, i\_e) 跨越了辅音，合成逻辑在网格中难以体现（如 c-a-k-e）。
* **简化规则：元音组合打包掉落。**
* **具体操作：** 将 **a\_e, i\_e, o\_e** 作为一个完整的“高级元音块”掉落。同时，ee, ea, ai 等也作为整体棋子掉落。
* **合成公式示例：**
  + **c + a\_e + k = CAKE** (蛋糕炮台)
  + **t + r + ee = TREE** (树人守卫)

### Level 4: Consonant Blends (辅音连读阶段)

* **痛点：** 如果让玩家用 s 和 h 合成 sh，或者 b 和 l 合成 bl，会严重占用格子且拖慢节奏。
* **简化规则：Digraphs (字母组合) 和 Blends (连缀) 视为单一特殊棋子。**
* **具体操作：** **sh, ch, th, wh, bl, cr, st** 等在Level 4关卡中，作为单个独立的高级棋子从召唤池中刷出。
* **合成公式示例：** **sh + i + p = SHIP** (战舰支援)；**f + r + o + g = FROG** (青蛙法师)。

### Level 5: Letter Combinations (进阶字母组合与词缀阶段)

* **具体操作：** 同Level 4的逻辑，高级的控制元音 (**ar, er, ir, or, ur**) 以及双元音 (**ou, ow, oi, oy**) 全部打包为“核心能量块”棋子掉落。
* **合成公式示例：**
  + **s + t + ar = STAR** (星星激光塔)
  + **c + oi + n = COIN** (金币穿透塔)

## 四、 “卡牌构筑 (Deck Building)” 机制：完美解决规则交叉

即使使用了简化规则，召唤池中依然会有大量无用的字母。为了提高合成成功率和游戏流畅度，引入“局外卡组配置”功能：

| 机制名称 | 机制详解与教育作用 |
| --- | --- |
| **赛前选词 (Word Deck)** | 每局游戏开始前，玩家最多只能选择 **4-5个目标单词英雄**（例如：选定 cat, dog, pig, cake, star）。 |
| **召唤池收缩** | 游戏开始后，消耗“智慧果”进行召唤时，**系统只会掉落这5个目标单词所包含的字母或词根块**。不会掉落任何无用的杂乱字母。 |
| **消除交叉冲突** | 如果玩家选了 cat 和 can。系统掉落了 c, a, t, n。玩家既可以用 c+a+t 合成猫咪，也可以用 c+a+n 合成罐头。一切由玩家拖拽决定，不会出现系统自动误合的情况。 |
| **学习聚焦** | 设计者/家长可以利用此机制，将每一关设定为特定的复习单元（例如：Unit 1 考核关卡，只能携带 Unit 1 的词汇参战），做到真正的靶向教学。 |

## 五、 总结与延时决定事项

上述规则将繁杂的拼读体系转化为游戏内的 **(单字母) + (发音组合块) = (单词)** 的简单逻辑。这种模块化(Chunking)设计是国际主流的自然拼读教学法。对于**“词缀如 -tion, -sion”**是否需要加入战斗，建议由设计者延时至后期决定，可以在前期先以 Level 1-3 的高频实体词汇（动物、食物、物品）作为游戏核心，验证塔防爽感后再逐步开放Level 5的抽象词汇。
