# 《拼读保卫战》IAA游戏原型开发设计文档

## 一、 项目概述与架构选型

本项目旨在开发一款寓教于乐的自然拼读塔防小游戏，针对学习牛津自然拼读1-5级的儿童。游戏结合了塔防策略与单词拼读合成机制，通过手动拖拽合成，加深孩子对自然拼读规则的记忆。

### 1. 架构选型与开发策略

* **前端框架：** Next.js (React) + Tailwind CSS。采用纯DOM驱动，AI友好度极高，适合快速验证原型。
* **视觉风格：** 超大号UI，系统自带大号 Emoji 为主（零美术成本，儿童易接受），若无对应Emoji则显示加粗单词。
* **后端/存储：** Supabase (原型后期接入)。用于存储不同等级的词汇库、掉落规则及合成公式，实现动态配置。
* **商业化/激励机制：** 原型阶段采用“Super Simple Songs 激励时间”替代视频广告。玩家遭遇困境触发30秒儿歌播放，结束后发放奖励。

## 二、 核心玩法与合成机制（手动拖拽叠加）

为了契合教育目的（拼读 Blending），游戏采用**手动拖拽到指定格子确认合成**的方式，而非碰撞自动消除。整个游戏可视为一个受前端渲染驱动的状态机。

### 1. 核心状态数据结构定义（给AI的参考）

// 网格状态 (3行 x 5列 Droppable 区域)
type Cell = {
 id: string;
 type: 'empty' | 'letter' | 'rime' | 'hero';
 value: string; // 例如: 'c', '-at', 'cat'
 emoji?: string; // 例如: '🐱'
}
// 备选区状态 (Draggable 字母块)
type HandItem = { id: string, type: 'letter' | 'rime', value: string }
// 敌人状态
type Enemy = { id: string, row: number, x: number, hp: number, emoji: '👾' }
// 子弹状态
type Bullet = { id: string, row: number, x: number, damage: number, emoji: '🐟' }

### 2. 合成规则与交互逻辑

* **超大尺寸UI：** 格子需达到 w-24 h-24，字母字体 text-5xl。极度宽容的容错吸附。
* **同质升级：** 拖拽相同字母块合并（如 'a' + 'a' = 2级 'a'），提升基础属性。最高5级。
* **异质合成 (单词英雄)：** 采用**Chunking (模块化)** 简化规则。例如：将辅音 'c' 拖拽至包含词根 '-at' 的格子中，触发字典校验。若匹配 {'c+-at': {word: 'cat', emoji: '🐱'}}，则格子转变为英雄状态，播放跳动动画。若无Emoji则显示 **CAT**。

## 三、 塔防循环系统 (Game Loop)

采用类似《植物大战僵尸》的单行防守模式，降低原型复杂度。

* **路线设定：** 3行网格对应3条独立防守路线。
* **出怪机制：** 每隔X秒，在某一行最右侧(x=100%)生成向左移动的遗忘怪(👾)。
* **攻击机制：** 处于英雄状态的格子，定时向所在行右侧发射对应子弹（绝对定位移动）。
* **碰撞结算：** 子弹 x 坐标越过怪物 x 坐标且在同一行时，怪物扣血。
* *注：前端开发时需注意 React setInterval 中的闭包陷阱，建议使用 useRef 保存最新状态。*

## 四、 词汇进阶大纲 (Level 1-5 知识库)

以下词汇库数据将在原型最后一个阶段整体迁移至 Supabase，实现动态拉取。

### Level 1: The Alphabet (字母发音)

**设计策略：** 仅掉落26个单字母。重点在于同质升级。升满级后可变为代表性单词英雄（如 A -> Apple 🍎）。

| 单元 | 发音 | 核心词汇 |
| --- | --- | --- |
| Unit 1-3 | Aa-Ii | apple, bear, cat, dog, egg, fish, goat, hat, igloo |
| Unit 4-6 | Jj-Rr | jam, kite, lion, monkey, nut, octopus, panda, queen, rabbit |
| Unit 7-8 | Ss-Zz | sun, tiger, umbrella, van, wolf, fox, yo-yo, zebra |

### Level 2: Short Vowels (短元音 CVC)

**设计策略：** 打包词族(Word Family)整体掉落。如掉落辅音 'c' 和词根块 '-at'，降低凑齐难度。

| 单元 | 拼读家族 | 合成示例 |
| --- | --- | --- |
| Unit 1-2 | Short a (am, an, ad, ag, ap, at) | c + -at = **CAT** 🐱 |
| Unit 3 | Short e (ed, eg, en, et) | p + -en = **PEN** 🖊️ |
| Unit 4 | Short i (ig, in, ip, it) | p + -ig = **PIG** 🐷 |
| Unit 5 | Short o (og, op, ot, ox) | d + -og = **DOG** 🐶 |
| Unit 6 | Short u (ub, ug, up, ut) | b + -ug = **BUG** 🐛 |

### Level 3: Long Vowels (长元音与 Magic E)

**设计策略：** 跨越辅音的Magic E（如 a\_e, i\_e）和元音组合（ee, ai）作为完整“高级元音块”掉落。

| 单元 | 规则 | 合成示例 |
| --- | --- | --- |
| Unit 1 | a\_e | c + a\_e + k = **CAKE** 🎂 |
| Unit 2 | i\_e | k + i\_e + t = **KITE** 🪁 |
| Unit 3 | o\_e | b + o\_e + n = **BONE** 🦴 |
| Unit 5 | ee, ea | t + r + ee = **TREE** 🌳 |

### Level 4: Consonant Blends (辅音连读)

**设计策略：** 连缀和组合（sh, ch, bl, cr）视为单一特殊字母块掉落。

| 单元 | 组合 | 合成示例 |
| --- | --- | --- |
| Unit 1 | L-Blends (bl, cl, fl) | cl + o + ck = **CLOCK** 🕰️ |
| Unit 2 | R-Blends (br, cr, fr) | fr + o + g = **FROG** 🐸 |
| Unit 4 | Digraphs (sh, ch, th) | sh + i + p = **SHIP** 🚢 |

### Level 5: Letter Combinations (进阶组合)

**设计策略：** 控制元音(ar, er, or)和双元音(ou, oi)打包为核心块掉落。

| 单元 | 组合 | 合成示例 |
| --- | --- | --- |
| Unit 1 | ar, ir, ur, er, or | s + t + ar = **STAR** ⭐ |
| Unit 2 | ou, ow, oi, oy | c + oi + n = **COIN** 🪙 |
| Unit 5 | Open Syllables | b + a + b + y = **BABY** 👶 |

## 五、 卡牌构筑机制 (解决规则交叉)

为避免召唤池中出现杂乱字母导致无法合成，采用“局外卡组配置”。

* 玩家单局仅能携带 4-5 个目标单词英雄（或通过配置指定复习单元）。
* 召唤池**仅掉落构成这几个目标单词所需的字母或组合块**。
* 若存在交叉（如带了 cat 和 can，掉落 c, a, t, n），由玩家手动拖拽决定合成结果。
